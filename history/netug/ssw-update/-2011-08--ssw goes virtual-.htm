Blind Refactoring - Extracting elegant domain models from ugly code bases, even if you don't understand the code

Speaker: Philip Laureano

Have you ever wondered if there was a better way to learn refactoring? Martin Fowler's Refactoring book was a great introductory book on how to clean up legacy code bases, but over the years, what I have found missing in that book is a set of guidelines that links all the different types of refactoring techniques together into one continuous process. In this talk, I will show you how to take almost any code base, and refactor everything from the simplest nested' 'if' blocks all the way to extracting an actual domain/object model to using an IOC container framework. Using a set of repeatable steps, I will show you how easy it is to refactor almost any code base, even if you don't understand every part of the application.

Session Outline:

Introduction:

  1. Accidental vs Essential Complexity
  2. Essential Complexity as a design tool
  3. Chipping away accidental complexity to reveal the actual design

Blind Refactoring in Ten Easy Steps

  1. Flattening nested if blocks
  2. Method extractions
  3. Class extractions
  4. The art of delegation
  5. Breaking the dependencies
  6. Recursive refactoring
  7. The tradeoffs of class decomposition vs inheritance, and how you can fix it
  8. Putting the pieces back together again / dealing with ravioli code
  9. The magic composition abilities of an IOC container
  10. The different flavors of IOC containers in .NET

About the Speaker:

Philip Laureano is the author of LinFu, an open source framework for implementing AOP, dynamic proxies, design by contract, mixins, inversion of control, and universal event handling on the Common Language Runtime. He is also the author of Hiro, the world's first and 'fastest' IOC container that is written in pure IL. On his spare time, you can usually find him hacking his own compiler and implementing the CLR Metadata specifications for fun. Philip has been professionally writing software in general since 1999, and coding in IL since 2005.
