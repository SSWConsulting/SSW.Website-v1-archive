Installation

To install the SSW Team Calendar, download and run the installation package from the SSW Team Calendar Home Page

The package will install into your default IIS "wwwroot" directory. You will need to read the following sections regarding how to configure the web page.

The installation will create a program group called "SSW Team Calendar" containing a link to the Team Calendar website and the Registration program.

You will have to complete a six part installation procedure.
Step 1: Creating a Team Calendar User on your Domain
Step 2: Giving Team Calendar permissions to users calendars
Step 3: Changing IIS permissions for TeamCalendar
Step 4: Verify you have the requirements for Team Calendar
           (Note: This page runs using http://localhost therefore you need to be running this
           from the Exchange Server)
Step 5: Run Team Calendar for the first time
           (Note: This page runs using http://localhost therefore you need to be running this
           from the Exchange Server)

           You can then open up and use the login credentials:
            User: Admin
            Password: Admin

Connection Methods

There are two different methods for connecting to the Exchange Server: HTTP and WebDAV.

The HTTP method needs to have access to all selected user's mailboxes. If it does not then the page will not load properly.

NOTE: the HTTP method is unsupported and will not work with all machines.

The WebDAV method needs to have Team Calendar installed on the same machine as Exchange Server 2000. You will need to specify the full Exchange Domain name in order for it to work properly.
