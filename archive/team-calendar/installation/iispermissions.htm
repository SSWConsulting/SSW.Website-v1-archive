Changing IIS permissions for TeamCalendar

Open up IIS by going Start -> Programs -> Administrative Tools -> Internet Services Manager

Right click on the TeamCalendar Virtual Web and go to properties.
Click on the "Directory Security" Tab and click on the "Edit" button under the "Anonymous access and authentication control" group.
Make sure Integrated Windows Authentication is unchecked.

The next step in Windows XP:

Now enter in the account details of the account you have previously created.

The next step in Windows 2000:

Click on the "Edit" button under the "Anonymous access" group.


Now enter in the account details of the account you have previously created.

What you have done is make any web pages under that virtual directory run under the 'SSWTeamCalendar' user rather than the IUSR_Annon user.

Changing IIS permissions for TeamCalendar

If you are to run Teamcalendar on the domain controller (exchange is usually on the domain controller) then you must give the Teamcalendar user permissions to log onto the server. To do this you need to:

  1. Open Group Policy by: Start -> Administrative Tools -> Domain Controller Security Policy
  2. Open the tree: Security Settings -> Local Policies -> User Rights Assignment

  3. On the right hand side list box, open the item Log on locally, and add TeamCalendar user

This step is now complete. Click 'Back' and select 'Step 4'.
